# Socrates 连续看不懂分支修正
Date: 2026-04-13
Status: implemented

## 本轮目标

- 补清主提示链里“学生一直说看不懂”时的处理逻辑。
- 明确区分：
  - 第一次看不懂
  - 同一步连续第二次看不懂
- 避免模型在同一个抽象问题上反复追问，导致对话方向跑偏。

## 已实现规则

### 1. 主链规则明确化

- 第一次说“看不懂 / 不会 / 没明白”：
  - 不整题重启
  - 先把当前问题缩小一点
  - 继续围绕当前最小卡点提问

- 同一步连续第二次还说“看不懂”：
  - 不重复上一问
  - 不继续追打同一个抽象问题
  - 回退一层
  - 重新锚定“已知 / 所求 / 当前障碍”
  - 改成更具体的观察、定位或判断动作
  - 从更小锚点重新开始，而不是改成长篇讲解

### 2. 分学科回退逻辑

- 数学：
  - 小学：优先回退到一个条件、一个数量、一个图形对象
  - 初中：代数回退到“已知 - 目标 - 最关键式子”；几何回退到“对象 - 已知关系 - 目标关系”

- 语文：
  - 回退到“题干任务 - 关键词 - 原文位置 / 文本锚点”
  - 阅读先抓一个词、一个句子、一个细节
  - 不直接跳去抽象主题或标准模板

- 英语：
  - 回退到“题型 - 局部语境 - 句子骨架”
  - 语法 / 完形优先看空前空后、词性、时态、固定搭配
  - 阅读先分清细节 / 主旨 / 推断，再定位原文

### 3. 产品方向对齐

- 不是“学生一说看不懂就从头再来”。
- 也不是“死盯上一问继续追问到学生更糊涂”。
- 当前实现采用的是：
  - 第一次先缩小
  - 第二次同一步还不懂就回退一层
  - 回退后继续保持苏格拉底式单步引导

这保证了方向不跑偏，同时避免模型变成整段讲题。

## 后续人工验收建议

- 用同一题连续两次输入“看不懂”，确认模型不会机械重复同一句提问。
- 数学题确认第二次回退后会回到已知、所求、对象或关系。
- 语文题确认第二次回退后会回到题干关键词和原文定位。
- 英语题确认第二次回退后会回到题型、局部语境和句子骨架。
- 确认回退后仍然只有一个问题，没有变成长篇讲解。
