# Socrates 学习闭环指导书大纲

> 日期: 2026-03-31
> 适用范围: `socra-platform / apps/socrates`
> 文档定位: 面向学生、家长、内部运营的配套指导书总纲
> 核心原则: 先讲学习闭环与思维体系，再讲产品页面与操作

---

## 1. 文档目标

这份指导书不是传统“功能说明书”，而是一份围绕 Socrates 当前产品能力整理出的学习闭环方法手册。

目标有三件事：

1. 说明 Socrates 不是单点解题工具，而是一套围绕“发现问题 -> 理解问题 -> 验证迁移 -> 间隔复习 -> 稳定掌握 -> 家长协同”的完整闭环系统。
2. 把当前已经实现的页面、功能、通知、任务、报告等能力，放回到同一条学习主线中解释。
3. 为后续扩写正式教程、家长手册、学生上手文档提供统一框架。

---

## 2. 文档受众

### 2.1 学生用户

- 想知道这套系统应该怎么用，而不是只知道每个按钮在哪里。
- 想理解“为什么做完题还要复习、还要做变式、还要看报告”。

### 2.2 家长用户

- 想知道自己在系统中的正确角色是什么。
- 想知道什么时候应该跟进、什么时候不该过度介入。

### 2.3 内部团队

- 需要统一对外表达“产品价值”和“学习闭环”的话术。
- 需要在产品迭代时始终围绕同一套方法论，而不是把页面功能越堆越散。

---

## 3. 写作原则

### 3.1 先方法论，后页面

- 先回答“为什么这样学”。
- 再回答“系统里怎么执行”。

### 3.2 先闭环，后功能

- 不按菜单结构写教程。
- 按学习动作、判断标准、角色协作顺序来组织内容。

### 3.3 先判断完成，再讲操作步骤

每一章都要明确：

- 这一阶段要达成什么目标
- 什么信号说明还没真正完成
- 系统里的哪些动作能帮助用户完成这一阶段

### 3.4 页面说明只做支撑，不做主线

- 页面路径、按钮位置、常见点击流程放到附录或操作索引。
- 主文重点写“思路、判断、闭环”。

---

## 4. 建议成品结构

建议最终不是一份超长文档，而是拆成三层：

1. 主指导书
2. 快速上手版
3. 操作附录

### 4.1 主指导书

用途：

- 讲清产品理念、学习闭环、角色协作、案例、常见误区

### 4.2 快速上手版

用途：

- 给新用户 10 分钟读完
- 快速知道如何跑通最小闭环

### 4.3 操作附录

用途：

- 放页面路径、功能映射、FAQ、异常排查

## 5. 主指导书一级框架

### 第一章: 产品定位与核心理念

建议回答：

- Socrates 到底是什么
- 它和题库、搜题、单次讲题工具的区别是什么
- 它真正想解决的学习问题是什么

建议关键词：

- 错题诊断
- 引导理解
- 迁移证据
- 间隔复习
- 稳定掌握
- 家长协同

### 第二章: Socrates 的完整学习闭环

这是整份指导书的主轴，建议画成一条完整流程：

1. 发现问题
2. 诊断问题
3. 引导理解
4. 迁移练习
5. 间隔复习
6. 稳定掌握
7. 家长跟进与支持

每一步都建议写四类内容：

- 这一环节的目标
- 学生应采取的动作
- 系统会给出的反馈
- 什么情况下不能算真正完成

### 第三章: 学生如何使用这套系统

建议从真实学习顺序来写，而不是从页面目录来写：

1. 登录并进入工作台
2. 上传题目或进入学习入口
3. 与 AI 对话暴露思路
4. 完成变式训练，形成迁移证据
5. 进入复习计划
6. 查看报告和掌握状态
7. 持续使用通知、任务和历史记录维持学习节奏

建议映射当前核心页面：

- `/workbench`
- `/study`
- `/error-book`
- `/review`
- `/reports`
- `/notifications`
- `/planner`
- `/knowledge`

### 第四章: 家长如何参与而不打断学习

建议强调家长的正确角色：

- 支持者
- 观察者
- 提醒者
- 协同者

不建议把家长写成：

- 替代学生思考的人
- 只看结果、不看过程的人
- 只在出问题时临时介入的人

建议覆盖当前能力：

- 建家庭
- 绑定孩子
- 下发任务
- 查看周报
- 查看提醒
- 进行必要干预

对应页面：

- `/family`
- `/tasks`
- `/controls`
- `/notifications`

### 第五章: 各核心功能在闭环中的作用

本章不做功能堆叠，而是解释“每个能力在闭环里承担什么职责”。

建议子章节：

1. 学习工作台
2. 错题本
3. AI 对话
4. 变式训练
5. 复习系统
6. 学习报告
7. 家长任务系统
8. 通知系统
9. 成就、积分与激励
10. 订阅与增强能力

每个子章节建议统一结构：

- 这个功能解决什么问题
- 它处在闭环中的哪一环
- 用户最容易怎么误用
- 正确使用后会看到什么信号

### 第六章: 一条完整案例演示

建议至少写 2 个案例：

1. 数学错题闭环案例
2. 英语或语文学习闭环案例

每个案例建议固定结构：

1. 初始问题
2. 学生第一次表现
3. 系统识别出的关键风险
4. 引导与练习过程
5. 迁移证据如何形成
6. 复习如何验证稳定性
7. 家长看到了什么、应该如何参与

### 第七章: 常见错误用法与纠偏

这一章建议直接写“不要这样用”。

示例主题：

- 把 AI 当答案生成器
- 做对一次就以为真正会了
- 只上传题目，不进入复习
- 只看结果，不看迁移证据
- 家长替学生完成思考
- 把通知当成噪音，不做动作

### 第八章: 不同阶段用户的使用策略

建议按成熟度分层：

- 新用户
- 轻度用户
- 持续使用用户
- 家长协同用户

每层都建议回答：

- 当前阶段最重要的目标是什么
- 最小应完成哪些动作
- 不必一开始就追求哪些高级能力

### 第九章: 日常执行建议

建议从节奏上给出使用建议：

- 每天做什么
- 每周做什么
- 什么时候该补复习
- 什么时候该做迁移练习
- 什么时候需要家长支持
- 什么时候说明不是“题太难”，而是“理解方式出了问题”

### 第十章: 附录与操作索引

附录只做支撑信息，不应盖过主文主线。

建议包含：

- 页面路径索引
- 功能与页面映射
- 常见问题
- 术语解释
- 异常记录模板

## 6. 各章节建议展开要点

### 6.1 产品定位与核心理念

建议重点讲清以下判断：

- 真正的学习问题不是“这道题做没做”，而是“学生能否迁移、能否稳定保留”
- 系统的核心目标不是一次做对，而是形成可重复、可追踪、可解释的掌握
- 只有把错题、对话、变式、复习、报告、家长协同串起来，才叫闭环

### 6.2 完整学习闭环

建议每个环节都标出完成条件：

- 发现问题: 有明确的问题进入系统
- 诊断问题: 不只是知道错了，还要知道错因
- 引导理解: 学生能讲出思路，而不只是看懂答案
- 迁移练习: 学生能在变式场景中再次正确完成
- 间隔复习: 学生能跨时间稳定复现
- 稳定掌握: 状态不再只是临时会，而是经过验证
- 家长协同: 家长的动作是支持闭环，而不是替代闭环

### 6.3 学生使用部分

建议用“动作 + 判断标准”的写法：

- 上传题目不是目的，进入学习状态才是目的
- 跟 AI 对话不是为了抄答案，而是为了暴露漏洞
- 做变式不是额外负担，而是验证迁移是否成立
- 复习不是补救，而是闭环的一部分
- 报告不是展示页面，而是复盘工具

### 6.4 家长使用部分

建议强调：

- 家长最重要的不是替孩子提高正确率，而是帮助孩子维持闭环
- 家长任务应围绕“跟进、提醒、支持、观察”
- 家长最值得关注的是风险信号和学习状态，而不是一次性的成绩波动

### 6.5 案例演示

建议每个案例都体现三条线：

- 学生线: 问题如何进入系统并被解决
- 系统线: 系统如何判断、提醒、推进
- 家长线: 家长在何时看见、何时动作、何时不介入

## 7. 快速上手版建议框架

快速上手版应比主指导书更短，适合首次接触用户。

建议结构：

1. 这套系统不是干什么的
2. 第一次使用只需要跑通哪 5 步
3. 学生最先用哪几个页面
4. 家长最先做哪几个动作
5. 什么信号说明你已经开始进入闭环

建议首次只聚焦以下路径：

- 学生: `/workbench` -> `/review` -> `/reports`
- 家长: `/family` -> `/tasks` -> 周报 / 通知

---

## 8. 操作附录建议框架

附录建议偏工具性，给已经理解闭环的人查用。

建议结构：

### 8.1 页面路径总表

- 学生侧页面
- 家长侧页面
- 登录与身份页面

### 8.2 功能到页面的映射

- 错题学习 -> `workbench` / `error-book`
- 复习 -> `review`
- 报告 -> `reports`
- 家庭管理 -> `family`
- 家长任务 -> `tasks`
- 通知 -> `notifications`

### 8.3 常见问题

- 为什么做对了还不是“真正掌握”
- 为什么还会收到复习提醒
- 为什么家长会收到跟进任务
- 为什么自定义域名异常时要先看 alias

### 8.4 异常记录模板

- 页面路径
- 账号角色
- 复现步骤
- 预期结果
- 实际结果
- 是否只在自定义域名出现
- 当前机器与网络信息

## 9. 建议优先写作顺序

如果要尽快落成第一版，不要平均铺开，建议按这个顺序写：

1. 第二章: 完整学习闭环
2. 第三章: 学生如何使用
3. 第四章: 家长如何参与
4. 第六章: 完整案例演示
5. 第七章: 常见错误用法与纠偏
6. 第十章: 附录与操作索引

原因：

- 先把主线写清，再补工具细节
- 先把角色协同写清，再补页面映射
- 先把案例写出来，整份指导书才会有真实感
