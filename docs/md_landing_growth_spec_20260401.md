## 一、目标

为 landing 建立一套不会破坏极简首页的自然流量增长系统。

一句话定义:

`用清晰的站点结构、持续连载的高质量文章、可被理解和引用的页面模板，以及适合国内搜索与分享习惯的承接路径，把自然流量做成复利引擎。`

---

## 二、总原则

### 1. 首页不为 SEO/GEO 变脏

不因为搜索优化而把首页做成:

- FAQ 农场
- 关键词堆积页
- 多模块信息垃圾场

首页继续保持极简，只承担:

- 主命题
- 主文章
- 系列结构
- 产品承接

### 2. 增长引擎主要发生在内容层

真正承接自然流量的核心页面是:

- `/essays`
- `/essays/[slug]`

首页主要负责品牌和总入口。

### 3. SEO 与 GEO 一套底层

对当前阶段最重要的不是分开做两套，而是共建这几个基础能力:

- 可抓取
- 可索引
- 可理解
- 可引用
- 可继续跳转

### 4. 国内优先

优先顺序:

1. 百度搜索
2. 微信搜一搜 / 微信分享回流
3. 夸克 AI 搜索
4. Kimi 联网搜索
5. DeepSeek 联网搜索
6. 豆包观察

---

## 三、增长架构

```text
Layer 1: Brand / Thesis
  /               首页
  /about          背景与可信度

Layer 2: Topic / Structure
  /essays         系列目录页

Layer 3: Query / Citation
  /essays/[slug]  单篇长文页

Layer 4: Conversion
  /start          产品承接页
```

逻辑:

```text
搜索 / AI 搜索 / 微信分享
  ->
首页或文章
  ->
继续阅读
  ->
理解体系
  ->
进入 Socrates
```

## 四、页面级增长职责

### 首页 `/`

职责:

- 解释站点在讲什么
- 给出主文入口
- 告诉用户这是一套体系
- 把流量导向深页

服务入口:

- 百度品牌词 / 主题词
- 微信品牌承接
- AI 搜索对品牌主题的理解

必须具备:

- 清晰 H1
- 主题鲜明的 metadata
- 指向 `/essays` 与主文的明显链接

### `/essays`

职责:

- 作为“书的目录页”
- 组织 `道 / 法 / 术 / 器`

服务入口:

- 百度目录型索引
- AI 搜索对系列结构的理解

必须具备:

- 系列说明
- 推荐起始文章
- 内链到各篇文章

### `/essays/[slug]`

职责:

- 承接具体问题搜索
- 成为 AI 搜索可能引用的答案源
- 让用户愿意继续读下一篇

服务入口:

- 百度问题词
- 夸克 / Kimi / DeepSeek 联网搜索
- 微信分享打开页

必须具备:

- 强问题标题
- 开头 2 到 3 段先给结论
- 清楚小标题结构
- 文末总结
- 下一篇
- Start 承接

### `/about`

职责:

- 表达作者、背景、方法来源
- 提升可信度

### `/start`

职责:

- 内容到产品的极简承接
- 不做第二个复杂 landing

## 五、国内引擎矩阵

### P0: 百度搜索

定位:

- 主收录引擎
- 主索引入口
- 品牌与问题词核心流量源

必须执行:

1. 百度搜索资源平台验证
2. 提交 Sitemap
3. 普通收录
4. 自动推送 / API 提交
5. 站点属性与品牌展现配置
6. 严格按移动落地页体验规范优化

### P0: 微信搜一搜 / 微信分享

定位:

- 品牌词承接
- 文章分享回流
- 微信生态曝光

必须执行:

1. 统一品牌命名
2. 配置微信分享卡片
3. 网站与公众号承接关系明确
4. Footer / About 保留官方微信触点

### P1: 夸克 AI 搜索

定位:

- 中文 AI 搜索引用源

策略:

- 不假设有站长平台
- 按“答案源”优化

### P1: Kimi 联网搜索

定位:

- 研究型 AI 搜索引用源

策略:

- 强定义
- 强结论
- 强方法结构

### P1: DeepSeek 联网搜索

定位:

- 中文联网回答引用源

策略:

- 强问题
- 强结论句
- 强页面可信度

### P2: 豆包

定位:

- 观察位

策略:

- 不单独定制
- 维持高质量公开页面即可

## 六、内容引擎

### 母题

- 从错误开始
- 闭环学习

### 系列

- 道
- 法
- 术
- 器

### 首批文章建议顺序

1. 粗心，是关闭追问的借口
2. 做对一次，不等于学会了
3. 为什么大多数错题本都是死书
4. 5 Why 如何用在学习上
5. PDCA 学习闭环
6. Socrates：把闭环电子化的工具

### 内容节奏

建议:

- 首批先上线 1 篇主文
- 之后每周 1 篇
- 每篇进入系列结构

---

## 七、文章模板要求

每篇文章页都必须满足以下要求。

### 内容结构

1. H1
2. 开头结论段
3. H2 / H3 结构化正文
4. Key Takeaways
5. Next in Series
6. Product Bridge

### 中文写法要求

1. 段落短
- 2 到 4 行为主

2. 标题更问题化

3. 结论句更直给

4. 小标题更清晰

5. 文末必须收束

### AI 搜索友好要求

1. 有定义句
2. 有判断句
3. 有结构化总结
4. 有作者背景和上下文

---

## 八、技术要求

### 必做

1. `sitemap.xml`
2. `robots.txt`
3. 页面级 title / meta description
4. Organization / WebSite structured data
5. Article / Breadcrumb structured data
6. canonical
7. 统一内链

### 强烈建议

1. IndexNow
2. 百度自动推送
3. 搜索控制台 / 百度资源平台监控

---

## 九、国内本地化要求

### 阅读习惯

1. 移动优先
2. 中文排版要易扫读
3. 首段直接说明问题

### 信任习惯

必须交代:

- 工程师爸爸背景
- 方法来源
- 真实场景
- ICP / 隐私 / 条款

### 分享习惯

要支持:

- 微信转发
- 微信收藏
- 朋友圈二次打开

不建议:

- 首屏大二维码
- 多个微信入口抢注意力

---

## 十、内链规则

内链是自然流量引擎的骨架。

必须保证:

### 首页

- 首页 -> 本周主文
- 首页 -> `/essays`
- 首页 -> `/start`

### `/essays`

- `/essays` -> 每篇文章
- `/essays` -> 推荐起始文章

### 单篇文章

- 文章 -> 下一篇
- 文章 -> 所属系列
- 文章 -> `/start`

---

## 十一、禁止动作

开发和内容阶段都不要做这些事:

1. 首页堆关键词段落
2. 首页加大 FAQ 区刷词
3. 做大量薄内容页
4. 文章模板化拼接
5. 太多 CTA 竞争
6. 用 SEO 破坏阅读体验
7. 用 GEO 名义制造无意义问答段

---

## 十二、成功指标

第一阶段先看:

1. 百度已索引页面数
2. 搜索曝光量
3. 搜索点击量
4. 首页 -> 主文点击率
5. 主文 -> `/essays` / `/start` 转化率
6. 文章页停留与滚动深度

---

## 十三、实施顺序

### Phase 1

- 首页重构
- 埋首页 metadata 与内链

### Phase 2

- `/essays`
- 第一篇文章详情页
- 文章模板 SEO / GEO 基础

### Phase 3

- `/about`
- `/start`
- 微信 / 百度接入细节

### Phase 4

- 迭代内容节奏
- 做引擎表现复盘
