# Socrates 几何链恢复收口记录

- 日期：2026-04-13
- 阶段：几何识别恢复与最小验收版收口
- 目标：在不改动主提示词链的前提下，补齐几何识别失败/低置信度时的恢复体验，避免用户当前图形和解题进度被覆盖。

## 本轮改动

### 1. 增加几何 UI 状态机

文件：
- `apps/socrates/app/(student)/workbench/page.tsx`

新增状态：
- `idle`
- `ready`
- `low-confidence`
- `unknown`
- `error`
- `preserved-unknown`
- `preserved-error`

作用：
- 将“识别成功”“低置信度”“未识别”“识别异常但已保留旧图”明确区分，避免前端只靠 `geometryData.type` 做粗判断。

### 2. 识别失败时保留上一版可用图形

本轮重点修复：
- 如果本次 `/api/geometry` 返回 `unknown`
- 或接口失败 / 超时 / 非 200
- 但页面上已有上一版可用几何图

则：
- 不覆盖原有 `geometryData`
- 不清空当前图形
- 保留用户当前可继续使用的图
- 页面改为显示“已保留上一版图形”提示

价值：
- 避免用户已经补好的辅助线、阅读进度、当前观察对象被错误冲掉。

### 3. 补齐 unknown / error 的前端分流

当前行为：
- `unknown` 或 `error` 时，显示更明确的黄提示卡
- 告知用户两条路径：
  - 无图题：直接继续答题
  - 有图题：先补 OCR 文本中的点、线、角条件后再重试

补充：
- 新增“继续答题”快捷按钮
- 重试按钮改成更明确的几何重试入口

### 4. 低置信度与保留旧图时增加提示条

当前在几何画板上方新增黄提示条，区分：
- 低置信度：结果可先辅助解题，但建议核对关键条件
- `preserved-unknown`：本次没识别出新图，已保留上一版图形
- `preserved-error`：本次识别异常，已保留当前图形

目的：
- 告诉用户现在看到的图是否可靠
- 告诉用户下一步该做什么

## 验收建议

### 场景 1：正常识别

预期：
- 数学题正常识别出图形
- 画板正常显示
- 顶部徽标显示“已识别”

### 场景 2：低置信度识别

预期：
- 画板仍显示
- 顶部徽标显示“待确认”
- 画板上方出现“建议先核对关键条件”的黄提示条

### 场景 3：先识别成功，再触发一次差结果

建议操作：
- 先用一题能稳定识别出图形的题目进入画板
- 再把 OCR 文本改坏一些，触发重新识别

预期：
- 原画板不被清空
- 出现“已保留上一版图形”或“已保留当前图形”的提示
- 仍可继续答题

### 场景 4：本身无图题

预期：
- 不显示画板
- 显示黄提示卡
- 可直接点“继续答题”
