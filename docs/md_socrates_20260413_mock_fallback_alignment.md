# Socrates Mock Fallback 对齐
Date: 2026-04-13
Status: implemented

## 本轮目标

- 对齐 `chat` 路由里的 fallback / mock 响应逻辑。
- 避免在以下场景下退回旧的泛化引导风格：
  - 本地无模型 key
  - 模型调用失败后自动降级

## 已完成内容

### 1. fallback 识别连续困惑

- 新增对“看不懂 / 不会 / 不明白”等表达的统一识别
- 支持判断：
  - 当前轮困惑
  - 连续第二次困惑

这意味着 mock 模式下也会区分：

- 第一次看不懂：先缩小一步
- 第二次连续看不懂：回退一层重新锚定

### 2. fallback 增加分学科兜底问法

- 数学：
  - 普通题回到已知 / 所求
  - 几何题回到点、线、角等对象观察

- 语文：
  - 回到题干关键词
  - 连续困惑时回到原文句段定位

- 英语：
  - 回到题型、局部语境、空前空后
  - 阅读题回到细节 / 主旨 / 推断判断

### 3. fallback 不再给旧式泛化话术

- 不再默认输出“万能鼓励 + 泛泛问下一步”
- 改为更接近当前主链风格：
  - 轻诊断
  - 一次一个问题
  - 连续困惑时回退
  - 学科差异化更明显

## 触达文件

- `apps/socrates/app/api/chat/route.ts`

## 本轮收益

- 本地验收时，即便模型 key 缺失，也更容易看到接近真实链路的引导行为。
- 模型降级时，体验不至于和当前主链规则明显脱节。
- 后续排查 prompt 问题时，不会被旧 mock 话术干扰判断。

## 建议回归点

- 关闭模型 key 或制造一次模型调用失败，确认会进入 mock。
- 在 mock 模式下连续两次输入“看不懂”：
  - 数学应回退到条件 / 所求 / 图形对象
  - 语文应回退到题干关键词 / 原文定位
  - 英语应回退到题型 / 局部语境 / 空前空后
