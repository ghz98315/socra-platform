- ✅ 顶部导航新增状态统计：已采纳 / 稍后处理 / 待处理
- ✅ 新增“下一条待处理”跳转逻辑，适合连续审阅而不是来回手动查找

- 为后续的“采纳持久化”和“移动端专用批注布局”打基础

### v1.4.0 - 原文批注工作台 + 审阅流 (2026-03-13)
- ✅ 扩展 `highlights` 数据结构，新增 `anchorText`
- ✅ 同步更新 system prompt、前端类型与归一化逻辑，要求亮点必须给出可挂载到正文的原文锚点
- ✅ 结果页新增“原文批注工作台”，支持把错别字、闪光点、金句、段落升级直接映射到学生原文
- ✅ 正文区支持类似 Word 的下划线高亮、编号标记、筛选显示（全部 / 错别字 / 闪光点 / 金句 / 段落升级）
- ✅ 新增侧边批注栏，点击正文高亮可自动定位对应批注；点击侧边批注可反向定位原文
- ✅ 新增批注导航条，支持按编号快速跳转、上一条 / 下一条连续审阅
- ✅ 新增悬停联动，正文与侧边批注可同步强调当前项
- ✅ 新增已阅状态与审阅进度条，支持用户按顺序查看完整批注集
- ✅ 新增键盘快捷键：`←/→` 或 `K/J` 切换批注
- ✅ 正文区升级为“纸面批改”视觉，增强老师批卷感而非普通结果卡片感

**本轮设计目标**
- 不是把分析结果堆在作文结尾，而是让“批改意见”直接附着在学生原文上
- 让家长/学生一眼看到“哪里好、哪里错、哪里该改”，降低理解成本
- 把结果页从“AI 报告”提升为“可连续审阅的批改界面”

**当前实现边界**
- 已完成正文文本级批注联动，前提是模型返回的 `anchorText` / `original` / `sentence` 能在 `body` 中逐字匹配
- 当前尚未完成“原图坐标级批注”，即还不能直接在学生手写图片上画出精确批注点位
- 若要继续升级到“原图批注映射”，需要 OCR 输出句段级或词级坐标

**建议下一步**
1. 为 OCR / 转写结果补充段落或句子坐标，打通“原图 -> 转写正文 -> 侧边批注”三层联动
2. 为批注项增加“采纳/忽略”状态，形成真正的批改闭环
3. 评估移动端的批注工作台布局，必要时切为底部抽屉式批注栏

### Phase 2: 数据持久化 (v1.2.0) ✅ 完成

**目标**: 保存作文批改历史

**数据库表设计**:
```sql
-- essays 作文表
CREATE TABLE essays (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  profile_id UUID REFERENCES profiles(id),
  title VARCHAR(200),
  content TEXT NOT NULL,
  grade VARCHAR(50) NOT NULL,
  images JSONB,
  analysis JSONB,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```
