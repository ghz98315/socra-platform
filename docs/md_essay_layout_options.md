# 作文结果页桌面端改版方案对比

## 背景

当前桌面端结果页里，`原文批注工作台` 被放在左侧主列内部，正文区还要再和右侧批注栏二次分栏，导致：

- 作文正文可读宽度偏小
- 首屏信息密度偏高，批改区域显得乱
- 用户进入页面后，视线容易被多块卡片同时拉扯

参考图：`D:\github\Socrates_ analysis\01.png`

目标：

- 让作文正文重新成为页面主角
- 保留“Word 式批注工作台”的沉浸感
- 让首屏信息层级更清楚，减少拥挤感

---

## 方案 A：中轴正文优先

### 适用方向

- 最强调“作文正文是主角”
- 最强调“批注工作台感”
- 允许评分、老师总评等分析卡片下沉到第二屏

### ASCII 设计图

```text
┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│ 顶部工具条                                                                                  │
│ 作文标题 | 年级 | 当前筛选 | 导出 | 下一篇 | 审阅进度                                       │
└──────────────────────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────┬──────────────────────────────────────────────┬──────────────────────┐
│ 左侧资料栏           │ 中央正文主区                                 │ 右侧批注工作台       │
│ 260~300px            │ minmax(760px, 1fr)                           │ 320~360px            │
│                      │                                              │                      │
│ ┌──────────────────┐ │ ┌──────────────────────────────────────────┐ │ ┌──────────────────┐ │
│ │ 原图预览         │ │ │ 作文标题                                 │ │ │ 当前批注卡       │ │
│ │ 缩略图切换       │ │ ├──────────────────────────────────────────┤ │ │ 类型/状态/序号   │ │
│ └──────────────────┘ │ │ 筛选 chips + 上一条/下一条 + review条    │ │ │ 对应原文         │ │
│                      │ ├──────────────────────────────────────────┤ │ │ 采纳/稍后/清除   │ │
│ ┌──────────────────┐ │ │                                          │ │ └──────────────────┘ │
│ │ 批改总览         │ │ │             作文纸面正文                  │ │                      │
│ │ 错别字/闪光点    │ │ │   下划线高亮、编号、锚点映射全部在这      │ │ ┌──────────────────┐ │
│ │ 金句/段落升级    │ │ │                                          │ │ │ 批注队列         │ │
│ └──────────────────┘ │ │                                          │ │ │ 可滚动列表       │ │
│                      │ │                                          │ │ │ 当前项高亮       │ │
│                      │ └──────────────────────────────────────────┘ │ └──────────────────┘ │
└──────────────────────┴──────────────────────────────────────────────┴──────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│ 第二屏分析区                                                                                 │
│ 等级评定 | 闪光点 | 金句 | 段落升级 | 老师总评 | 素材箱                                     │
└──────────────────────────────────────────────────────────────────────────────────────────────┘
```

### 优点

- 正文阅读面积最大
- 批注和正文关系最清晰
- 首屏任务单一，最像老师批改台
- 最符合“批注感秒杀竞品”的方向

### 风险

- 首屏的“报告感”会变弱
- 评分和老师总评下沉后，需要更强的首屏摘要来补足价值感

---

## 方案 B：双层工作台

### 适用方向

- 想保留“批改工作台 + 结果页展示”的平衡
- 不希望把评分和总评完全放到第二屏
- 想以更稳的代价改善正文宽度

### ASCII 设计图

```text
┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│ 顶部工具条                                                                                  │
│ 作文标题 | 年级 | 当前筛选 | 导出 | 下一篇 | 审阅进度                                       │
└──────────────────────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────┬──────────────────────────────────────────────┬──────────────────────┐
│ 左侧原图区           │ 中央正文区                                   │ 右侧批注区           │
│ 240~280px            │ minmax(680px, 1fr)                           │ 300~340px            │
│                      │                                              │                      │
│ ┌──────────────────┐ │ ┌──────────────────────────────────────────┐ │ ┌──────────────────┐ │
│ │ 原图预览         │ │ │ 作文标题 + chips + 导航                  │ │ │ 当前批注详情     │ │
│ │ 缩略图切换       │ │ ├──────────────────────────────────────────┤ │ ├──────────────────┤ │
│ │ OCR摘要          │ │ │                                          │ │ │ 批注队列         │ │
│ └──────────────────┘ │ │             作文正文                      │ │ │ 可滚动           │ │
│                      │ │                                          │ │ └──────────────────┘ │
│                      │ └──────────────────────────────────────────┘ │                      │
└──────────────────────┴──────────────────────────────────────────────┴──────────────────────┘

┌──────────────────────────────┬──────────────────────────────┬──────────────────────────────┐
│ 评分摘要卡                   │ 闪光点摘要卡                 │ 老师总评摘要卡               │
│ 分数/等级/简评              │ 3~4 条亮点                  │ 温暖鼓励 + 成长建议          │
└──────────────────────────────┴──────────────────────────────┴──────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│ 第三屏深度分析                                                                               │
│ 金句 | 段落升级 | 素材箱 | 完整老师总评                                                     │
└──────────────────────────────────────────────────────────────────────────────────────────────┘
```

### 优点

- 首屏既有批改感，也有结果展示感
- 家长/学生第一次打开更容易快速看到 AI 价值
- 改动范围相对更稳

### 风险

- 正文区虽然会变大，但不如方案 A 极致
- 首屏仍然会比方案 A 更满

---

## 建议

如果优先级是：

- 批注感
- 正文可读性
- 像真实老师批改

建议选 `方案 A`。

如果优先级是：

- 展示感
- 首屏完整度
- 更稳妥的改版节奏

建议选 `方案 B`。

---

## 推荐执行顺序

1. 先确认采用 `方案 A` 还是 `方案 B`
2. 第一轮只改桌面端整体栅格和首屏层级，不动移动端
3. 第二轮再细调正文宽度、批注栏宽度、卡片密度和留白
