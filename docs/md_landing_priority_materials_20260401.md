## 一、建议优先级

当前建议按下面顺序推进，不要跳步骤。

### P0: 先定首页骨架

这一层最优先，因为它决定后面的文案、视觉和文章系统是否会返工。

需要先定的东西:

1. 首页到底是不是“书籍连载首页”
2. 首页是否采用 `Hero -> 母题 -> 主文 -> 系列结构 -> 方法 -> 系统承接 -> CTA`
3. 产品区是否降级为“方法的数字化承接”
4. 首页的两个主动作是否固定为:
   - `阅读主文`
   - `免费体验`

如果这一步不先定，后面的文案和设计会继续摇摆在:

- 产品首页
- 内容首页
- 个人主页

之间。

---

### P1: 再定首页文案系统

骨架确定后，第二优先级是文案系统。

这一层要解决:

1. 首页标题到底写产品，还是写母题
2. “从错误开始”怎么写得既像书，也像首页
3. Socrates 在首页出现的位置和语气
4. 每一块模块的语气是否统一

建议这一层只解决文案框架，不急着进入视觉细节。

---

### P2: 再定文章系统信息架构

这一步决定内容会不会越写越乱。

需要明确:

1. 首页如何展示主文和最近文章
2. `/essays` 或 `/articles` 列表页怎么组织
3. `道 / 法 / 术 / 器` 如何成为分类结构
4. 文章详情页如何连接:
   - 下一篇
   - 模板
   - 产品体验

如果这一步不定，首页即使改漂亮了，后面文章一多也会失控。

---

### P3: 最后再进入视觉和代码实现

这一步才是具体设计和开发。

应该包括:

1. landing 视觉语言收敛
2. 首页模块落地
3. 文章列表页和详情页实现
4. 内容数据结构实现

## 二、当前最值得先确认的资料

如果只看当前阶段，最需要的资料就是下面 3 份。

### 资料 1: 首页主结构

最重要。

建议采用:

```text
Hero
  ->
Book Thesis
  ->
Featured Essay
  ->
Series Map
  ->
Socra Method
  ->
Product Bridge
  ->
Final CTA
```

### 资料 2: 内容母题

从 `article-outline.pdf` 提炼后，首页必须统一围绕这 4 句话展开:

1. 错误不是终点
2. 粗心不是根因
3. 做对一次不等于学会
4. 真正有效的学习必须形成闭环

### 资料 3: 产品定位句

建议固定为:

`Socrates 不是先出现的主角，而是这套闭环方法的数字化执行工具。`

这句话很重要，因为它决定了产品在首页上的位置。

## 四、推荐的页面节奏图

如果从“用户心理路径”看，建议页面节奏如下:

```text

  ->
再建立信任
  ->

  ->
再说明方法
  ->

```

对应到页面:

```text
Hero
  ->
Book Thesis
  ->
Featured Essay
  ->
Series Map
  ->
Socra Method
  ->
Product Bridge
  ->
CTA
```

这比“功能堆叠式首页”更有叙事性。

## 七、结论

按当前优先级，最重要的不是马上做视觉，而是先锁死这 3 件事:

1. 首页是“闭环学习方法”的总入口
2. 文章是主轴，不是附属模块
3. Socrates 是方法的系统承接，而不是首页第一主角

一旦这 3 件事确认，后续文案、页面和代码实现都会稳定很多。
