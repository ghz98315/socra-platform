# Socrates Phase 2 错题闭环人工验收清单

日期：2026-04-11
范围：`Phase 2`
目标：验证 `error-book -> error detail -> review -> review/session` 的主动作语义已经收口，用户能始终看到明确的下一步
执行稿：`docs/md_socrates_phase2_error_loop_acceptance_execution_20260411.md`

## 0. 本轮验收目标

- 错题列表页能优先给出单一下一步，不让用户在“看详情 / 继续学习 / 继续复习”之间犹豫
- 错题详情页只保留一条主动作，辅助动作不再与主动作并列抢焦点
- 复习首页能区分“现在该复习”和“最近完成后可回看”
- 单题复习完成页不会并列过多出口，完成态和未完成态的动作语义一致

## 2. 最短验收顺序

如果这轮只想先验证主链是否可用，按下面顺序走即可。

### Step 1：错题列表的下一步

- [ ] 打开 `error-book`
- [ ] 观察顶部 `Next Step`
- [ ] 预期主动作只出现一个：
  - `继续复习`
  - 或 `继续学习`
  - 或 `看原题`
  - 或 `开始录题`
- [ ] 预期不再出现泛化词：
  - `看最新错题`
  - `回看详情`

### Step 2：从列表进入详情或复习

- [ ] 在列表中任选一条错题
- [ ] 如果该题已在复习中：
  - 主动作预期为 `继续复习`
- [ ] 如果该题仍在学习中：
  - 主动作预期为 `加入复习` 或 `继续学习`
- [ ] 如果该题已完成：
  - 主动作预期为 `看原题`

### Step 3：错题详情页主动作

- [ ] 打开 `error-book/[id]`
- [ ] 观察页内 `Next Step` 卡
- [ ] 预期卡片只强调 1 条主动作
- [ ] 预期头部不再并列显示：
  - `继续学习`
  - `继续复习`
  - `加入复习`
  这些与主动作重复的按钮

### Step 4：复习首页主动作

- [ ] 打开 `review`
- [ ] 如果当前有待复习任务：
  - 预期顶部主动作是 `继续复习`
  - 辅助动作是 `看原题`
- [ ] 如果当前没有待复习但有最近完成记录：
  - 预期顶部主动作是 `看复盘`
  - 辅助动作是 `继续学习`

### Step 5：单题复习完成态出口

- [ ] 打开 `review/session/[id]`
- [ ] 完成一次判定，进入完成态结果页
- [ ] 如果本轮未关单：
  - 预期只保留 `继续学习 + 复习中心`
- [ ] 如果本轮已关单：
  - 预期只保留 `复习中心 + 看原题`
- [ ] 预期完成态不再并列三条出口

## 3. 详细人工验收清单

### 3.1 错题列表页 `error-book`

- [ ] 顶部 `Next Step` 卡存在且只有 1 个主按钮
- [ ] 辅助按钮固定为 `录新题`
- [ ] 列表卡片中的主动作与题目状态一致：
  - 已有复习记录时为 `继续复习`
  - 学习中时为 `加入复习` 或 `继续学习`
  - 已完成时为 `看原题`
- [ ] 列表卡片中的辅助按钮统一为 `看原题`
- [ ] 页面内不再混用：
  - `看详情`
  - `回看详情`
  - `看最新错题`

### 3.2 错题详情页 `error-book/[id]`

- [ ] 页内存在 `Next Step` 卡
- [ ] `Next Step` 卡主动作与当前状态一致：
  - 未完成题为 `继续学习`
  - 已进入复习为 `继续复习`
  - 已完成且已有复习记录为 `看复盘`
  - 已完成但无复习记录为 `复习中心`
- [ ] 头部保留的是辅助操作，不再重复出现主链按钮
- [ ] 头部 `导出PDF` 可以点击，不是禁用占位按钮
- [ ] 已完成态主出口不再回到 `error-book`

### 3.3 复习首页 `review`

- [ ] 顶部 `Next Step` 卡始终只有 1 个主动作和 1 个辅助动作
- [ ] 有待复习题时：
  - 主动作是 `继续复习`
  - 辅助动作是 `看原题`
- [ ] 无待复习但有最近完成记录时：
  - 主动作是 `看复盘`
  - 辅助动作是 `继续学习`
- [ ] 顶部右上角入口文案已统一为 `继续学习`
- [ ] “现在该复习 / 后续计划 / 最近完成”三块里的卡片动作词统一为：
  - `继续复习`
  - `看原题`
  - `看复盘`

### 3.4 单题复习页 `review/session/[id]`

- [ ] 顶部存在统一的 `看原题` 辅助入口
- [ ] `intro` 阶段主按钮是 `继续复习`
- [ ] `judge` 阶段提交后能进入完成态
- [ ] 完成态结果页：
  - 未关单时只显示 `继续学习 + 复习中心`
  - 已关单时只显示 `复习中心 + 看原题`
- [ ] 完成态不再同时显示：
  - `继续学习`
  - `复习中心`
  - `看原题`
  三者并列

### 3.5 题目恢复与工作台承接

- [ ] 从 `error-book` 或 `review` 进入 `study/[subject]/problem?session=...`
- [ ] 预期会直接进入 `workbench`
- [ ] 预期没有中间停留桥页
- [ ] 旧会话恢复后：
  - 有历史对话时，聊天区正常显示历史消息
  - 无历史对话时，至少显示一条开场 assistant 引导消息
- [ ] OCR 识别成功后不会明显出现同一题的重复几何解析闪动

## 4. 建议验收用例

### 用例 A：学习中题目

- [ ] 打开一条 `guided_learning` 状态的错题
- [ ] 在列表页确认主动作是 `加入复习` 或 `继续学习`
- [ ] 打开详情页确认 `Next Step` 主动作是 `继续学习`
- [ ] 进入 `workbench` 后确认聊天区不为空

### 用例 B：已进入复习的题

- [ ] 打开一条已有 `review_id` 的错题
- [ ] 列表页主动作应为 `继续复习`
- [ ] 详情页主动作应为 `继续复习`
- [ ] 进入 `review/session/[id]` 后，顶部仍可 `看原题`

### 用例 C：已完成的题

- [ ] 打开一条已完成且已有复习记录的题
- [ ] 列表页主动作应为 `看原题`
- [ ] 详情页主动作应为 `看复盘`
- [ ] `review` 首页无待复习任务时，主动作应优先 `看复盘`

## 5. 重点风险观察项

- [ ] 详情页头部是否仍残留重复主动作按钮
- [ ] 已完成态是否仍错误跳回 `error-book`
- [ ] `review` 首页在无待复习任务时是否仍错误显示 `继续学习`
- [ ] `review/session` 完成态是否又回到三按钮并列
- [ ] `workbench` 恢复旧会话时是否仍出现聊天区空白

如果这轮只想先判断“Phase 2 主链已经基本可用”，只验下面 5 步。

- [ ] 预期顶部 `Next Step` 只有 1 个主按钮
- [ ] 预期主按钮文案只可能是：
  - `继续复习`
  - `继续学习`
  - `看原题`
  - `开始录题`

### Step 2：错题详情页

- [ ] 打开任一 `error-book/[id]`
- [ ] 预期页内 `Next Step` 卡只强调 1 条主动作
- [ ] 预期头部不再重复出现 `继续学习 / 继续复习 / 加入复习`

### Step 3：复习首页

- [ ] 打开 `review`
- [ ] 有待复习题时：
  - 主动作预期是 `继续复习`
  - 辅助动作预期是 `看原题`
- [ ] 无待复习但有最近完成记录时：
  - 主动作预期是 `看复盘`

- [ ] 未关单时，预期只保留：

### Step 5：工作台恢复

- [ ] 从 `error-book` 或 `review` 回到 `study/[subject]/problem?session=...`
- [ ] 预期直接进入 `workbench`
- [ ] 预期聊天区不是空白
